Ethereum Smart Contract Benchmark

A custom performance evaluation architecture built natively using Node.js, Ganache, and Ethers.js.

UG Research (CSET-306)

Research Context

1. Introduction & Objective

Blockchain platforms suffer from throughput bottlenecks. As EVM adoption grows, precise capacity planning is essential. The objective of this research is to construct a lightweight, headless benchmarking engine capable of stressing EVM chains directly via RPC, bypassing the overhead of heavyweight external frameworks.

2. Problem Definition & Gap

Existing tools (like official Hyperledger Caliper) demand complex Docker orchestrations and localized dependencies, restricting agile CI/CD integration. The Gap: A lack of serverless, self-contained benchmarking pipelines that can execute in cloud-runners and reliably publish static TPS telemetry globally.

3. Literature Review Context

Current literature emphasizes the necessity of isolated testnets (Dinh et al.) to prevent network noise from polluting metrics. This project adapts these established principles by leveraging ganache for mathematically sterile, in-memory execution boundaries, ensuring empirical validity of the captured TPS payloads.

4. Proposed Methodology

A fully automated CI/CD pipeline triggered by a Developer Push. A GitHub runner provisions Node.js and Ganache, compiles the SecureDataStorage contract, fires concurrent Promise.all() read/write barrages via ethers.js, serializes metrics into JSON, and statically deploys the payload to a Global CDN.

Project Logic & Architecture

1. Network Initialization

Instead of relying on an external testnet or a standalone desktop application, this project programmatically instantiates an in-memory Ethereum blockchain using ganache. By spinning up Ganache natively in the background context, we create a sterile, zero-latency environment tailored precisely for pure performance stress-testing.

2. Contract Deployment

The script reads the pre-compiled Application Binary Interface (ABI) and Bytecode of the SecureDataStorage smart contract directly from the filesystem. Using ethers.js, it dynamically signs and broadcasts a deploy transaction, awaiting the block inclusion to obtain the live, queryable contract address.

3. High-Intensity Workloads

Once deployed, the core logic assaults the network with a configured number of parallel transactions simulating a rush of concurrent clients. It fires asynchronous Javascript promises asynchronously utilizing Promise.all() to maximize Node.js event-loop throughput on the blockchain RPC endpoint without waiting for procedural network replies.

4. Visualization & Telemetry

To bring this to the web, I wrapped the benchmarking CLI script inside a lightweight Express.js HTTP API server. When you submit the benchmark from this UI, it leverages fetch to trigger the backend execution stream. The server clocks the precise millisecond deltas from initiation to the final block confirmation, calculates Transactions Per Second (TPS) & Queries Per Second (QPS), and responds with the payload for this client to render.

Engineering Evolution: How It Was Built

This project was not built in a single step. It is the culmination of three distinct architectural phases, transforming a basic terminal script into a globally distributed web application:

1 Engine Refactoring

We began with a raw Node.js script. I refactored custom-benchmark.js to isolate the Ganache transactions and securely return the TPS/QPS metrics as JSON instead of simply printing to the console.

2 Local API Servers

A terminal script isn't a modern web app. I built a persistent backend wrapper using Express.js and constructed this stunning TailwindCSS dashboard to trigger benchmarks dynamically via HTTP fetch requests.

3 Serverless Deployment

To host freely and globally on GitHub Pages (which strictly requires static files), I wrote a new snapshot compilation script that fully bypasses the live API server, creating a totally Static Serverless App!

Advanced Execution Pipeline

LIVE Full State Machine

Every state transition from git push to global CDN delivery — fully automated, zero human intervention.

1. Developer Push

Push new Smart Contract code or logic changes to the main branch on GitHub.

2. Trigger gh-pages.yml

GitHub Actions detects the push and spins up a fresh ubuntu-latest cloud runner automatically.

3. Checkout & npm install

Runner clones the repository, then installs all Node.js dependencies including Ganache and Ethers.js.

4. Boot In-Memory Ganache

A fresh in-memory Ethereum blockchain is programmatically instantiated — isolated, zero-latency, and sterile.

5. Deploy Smart Contract

ethers.js signs and broadcasts a deploy tx for SecureDataStorage, awaiting block confirmation to get the live contract address.

6. 50× Write Transactions

Fires 50 concurrent storeData() calls via Promise.all() to stress-test write throughput on the live network.

7. 50× Read Queries

Fires 50 concurrent getData() calls — pure RPC lookups with no block confirmation wait, measuring raw query speed.

8. Calculate TPS & QPS

Millisecond deltas are analysed to produce precise Transactions Per Second and Queries Per Second benchmark metrics.

9. Export results.json

All metrics are serialised and written as a static JSON data snapshot to /public/results.json in the repository.

10. Live on Global CDN 🌐

The polished UI and fresh benchmark data are committed and published to GitHub Pages — live, globally, sub-second.

CI/CD Snapshot Pipeline Architecture

1. Developer Push

You push new Smart Contract code or logic changes to the main branch on GitHub.

2. GitHub Actions Pipeline

A secure Ubuntu Node.js runner is spun up in the cloud, installing all dependencies and preparing the environment.

3. Headless Execution

The generate-static-benchmark.js script silently spins up an isolated Ganache net, deploys the contracts, and blasts it with 50 txs to compute precise TPS.

4. Static Global Deploy

The final data is saved to /public/results.json, and the entire polished UI dashboard is published instantly to GitHub Pages globally.
